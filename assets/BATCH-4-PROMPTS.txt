 PROMPT 1 — bonfire_combined_reference.png   (working file, not imported)
   [attach: cat_biscuit_front_neutral.png as style-only reference]
-----------------------------------------------------------------------------
Game environment sprite for a cozy hand-drawn doodle village sim. Match
the art style of the attached character sprite exactly — uneven warm
dark-brown hand-inked outline (2–4 px, weight varying mid-stroke), flat
slightly-desaturated fills, no gradients, no realistic shading, charming
wobbly hand-drawn shapes — but do NOT include the cat or any character.

Subject: a small lit bonfire. A low pile of 4–6 crossed logs (warm wood
browns #b9946a and #8a6a44, simple doodle logs with visible end rings)
with one cozy flame burning on top (rounded teardrop flame, main body
warm orange #f0a04b with a smaller inner tongue in soft yellow #f6c65a,
drawn with the same inked outline).

Layout and registration (critical — this image will be split into two
game layers): canvas 256 x 256 px, PNG RGBA, fully transparent
background. The log pile is anchored bottom-center, its base at 90% of
canvas height; the pile occupies the lower third of the canvas; the TOP
surface of the pile sits at 60% of canvas height. The flame's base sits
EXACTLY on that 60% line, centered; the flame tip reaches no higher than
15% from the canvas top (empty headroom above — the game bounces the
flame upward). Flame and logs must overlap ONLY at the flame base /
pile top so they separate cleanly.

No scenery, no ground, no grass, no smoke, no sparks, no stones circle,
no text, no watermark, no shadow, no border, clean alpha edges. One
bonfire only, centered.

 PROMPT 4 — env_tree.png
   [attach: cat_biscuit_front_neutral.png as style-only reference]
-----------------------------------------------------------------------------
Game environment sprite for a cozy hand-drawn doodle village sim. Match
the art style of the attached character sprite exactly — uneven warm
dark-brown hand-inked outline, flat slightly-desaturated fills, no
gradients, no shading beyond at most one flat tone — but do NOT include
the cat or any character.

Subject: ONE simple doodle tree, the game's only tree (code creates all
variety by flipping, leaning, scaling, and hue-shifting this single
image). A slightly crooked trunk in warm brown #8a6a44 with a single
soft rounded cloud-like canopy in sage green #7fa25e — one friendly
blob canopy, gently lumpy outline, no individual leaves, no branches
visible outside the canopy, no fruit.

Layout and registration: canvas 256 x 256 px, PNG RGBA, fully
transparent background. Trunk base anchored bottom-center at 90% of
canvas height (the ground line); tree total height ~200 px; canopy
centered over the trunk, roughly symmetrical (it will be mirrored in
code and must read the same flipped); ~10% margin on all sides, nothing
touching canvas edges.

No ground, no grass, no roots spreading wide, no scenery, no text, no
watermark, no shadow, no border, clean alpha edges. One tree only.

-----------------------------------------------------------------------------
 PROMPT 5 — env_stump.png          (EDIT — attach approved PROMPT 4 output)
-----------------------------------------------------------------------------
Edit the attached tree image: the tree has been chopped down. Replace it
with ONLY a short stump — the bottom of the SAME trunk, same thickness,
same position on canvas, same warm brown #8a6a44 and identical doodle
outline style — about 30 px tall, cut flat on top, with 2–3 simple
hand-drawn tree rings visible on the pale cut surface (light wood
#c7a97e). Everything else (upper trunk, canopy) fully erased to
transparency.
Canvas stays 256 x 256 px, PNG RGBA, transparent background; the stump
base remains exactly where the trunk base was, on the 90% ground line
(the game swaps tree → stump in place with no offset). No wood chips, no
axe, no scenery, no text, no shadow, clean alpha edges.
