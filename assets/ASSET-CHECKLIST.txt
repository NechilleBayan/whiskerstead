THE MODEL (locked 2026-07-20)
 One image per distinct THING. Everything else is code, in the spirit of
 the existing squash-stretch walk:
   ANIMATION = transform the single image (squish, bounce, tilt, wobble).
               No _a/_b frame pairs anywhere.
   QUANTITY  = stack/offset multiple draws of the same image
               (3 wood = the one wood image drawn 3 times).
   STATE     = compose reusable layers (bonfire lit = woodpile + fire
               layer bouncing on top; unlit = woodpile alone).
   VARIANTS  = code flip / lean / scale / tint of the one image
               (tree variety, owner-colored houses).

MARKERS   [ ] to generate · [GEN] generated, not imported · [ADDED] in repo
ENGINE    WIRED (2026-07-20): generic loader over assets/world/{2x,1x}
          covers every row below, keyed by filename; missing files fall back
          to the procedural drawings per-file. To import art: pass the §11
          acceptance checklist, then drop the PNG into assets/world/2x/ and
          1x/ under its canonical name — the game picks it up on reload.
          Registration offsets are named constants atop canvas-renderer.ts.
          (decal_path loads but has no draw site yet — the sim has no path
          data; prop_rod/ladle/book load, held-prop poses are future work.)
SIZES     buildings author 512 (pond 768) · layers 256 · items 128.
          Export 2x + 1x, same filename, per SPRITE-SPEC-REFERENCE.txt.
